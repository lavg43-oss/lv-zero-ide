Orchestrator

3,420 líneas de EventEmitter. Agent loop con streaming, retry con backoff exponencial (3 intentos), fallback chain entre providers, health check cada 60s, checkpoint cada 30s, validación de mensajes, y abort handling.

Core

Circuit Breaker

State machine CLOSED→OPEN→HALF_OPEN. Failure threshold: 5. Open timeout: 60s. Eventos: circuit_open, circuit_half_open, circuit_closed. Stats con últimos 5 errores. Wrapping automático de providers.

Core
🧠

LLM Client

Multi-provider con soporte nativo para DeepSeek (reasoning_content en streams). Provider map con lazy init. Health tracking: degradación tras 3 fallos consecutivos, auto-recuperación a los 5 min.

Core
🔧

Tool Call Repair

Pipeline 4-pass: flatten (aplanar anidamiento), scavenge (recuperar llamadas perdidas), truncation (reparar JSON truncado), storm (StormBreaker — reparación agresiva con análisis de esquema).

Core
📦

CacheFirstLoop

Sistema de caché prefix-stable inspirado en Reasonix. AppendOnlyLog, ImmutablePrefix, VolatileScratch. Deduplicación de respuestas del LLM para evitar procesamiento redundante y ahorrar tokens.

Core
💾

State Manager

Persistencia de sesión en JSON. Save/restore/clear. Checkpoints automáticos. Recuperación ante crashes con detección de heartbeat stale (30s). Exportación de sesiones completas.

SIN COMPLICACIONES

💡 Cómo Funciona el Modelo Gestionado

1 El usuario paga $5 USD como mínimo (vía la plataforma que se integre — Stripe, PayPal, cripto, etc.)
2 El sistema convierte esos $5 en tokens DeepSeek (~4,400 llamadas combinadas Flash + Pro) — el usuario nunca ve montos en dólares, solo ve su saldo en tokens
3 La recaudación se divide internamente: ~40% va a tokens DeepSeek del usuario, el ~60% restante cubre mi API key Pro, infraestructura y margen del servicio
4 El usuario consume sus tokens llamada por llamada. El sistema descuenta tokens reales del balance. Cuando se acerca a 0, recibe notificaciones para recargar
5 Si el saldo llega a $0, el servicio se pausa automáticamente — sin cargos sorpresa, sin deudas. El usuario recarga cuando quiera y sigue desde donde se quedó

ARQUITECTURA TÉCNICA  Plan de Limitación de Consumo

📊

1. Sistema de Tracking por Usuario

Cada usuario tiene un balance en Supabase (tabla user_balances): user_id, api_balance (decimal), last_deduction, created_at. Cada llamada a DeepSeek descuenta tokens reales consumidos del balance del usuario.

CONTADOR EN DB
🛑

2. Hard Cap de Balance

Antes de cada llamada a la API, el sistema verifica: user.balance > 0. Si el balance es $0 o menor, la llamada se rechaza con "Saldo insuficiente — recarga para continuar". Sin deuda, sin sobregiro.

CORTE AUTOMÁTICO
📉

3. Umbral de Alerta Temprana

Cuando el balance baja de $0.30 USD, el sistema envía notificación al usuario: "Tu saldo está por agotarse. Quedan ~$X. Recarga para no interrumpir tu sesión." Segunda alerta a $0.10.

ALERTAS
⏱️

4. Rate Limiting por Usuario

Límite de requests por minuto por usuario (configurable, default ~10 req/min). Esto evita que un solo usuario abuse el sistema y asegura recursos equitativos entre todos los usuarios gestionados.

TOKEN BUCKET
🔐

5. API Key Rotativa (Seguridad)

La API key de DeepSeek no se expone al usuario. Se usa una key interna del servidor rotada periódicamente. El usuario nunca ve la key — solo consume su saldo. Esto previene fuga de keys y uso no autorizado.

SEGURIDAD
🔄

6. Persistencia de Sesión

Si el usuario queda en medio de un flujo de trabajo cuando el saldo se agota, el checkpoint se guarda automáticamente en Supabase. Al recargar saldo, la sesión se restaura exactamente donde se quedó — sin perder contexto.

SIN PÉRDIDA

📐 Proyección del Modelo

Cada recarga de $5 = ~4,400 llamadas DeepSeek (combinando 90% Flash + 10% Pro). Con uso moderado (~150 llamadas/día), eso es aproximadamente 1 mes de uso. Los montos en dólares son internos — el usuario solo ve su saldo en tokens.

💵 Usuario paga: $5 → solo ve tokens
⚡ Interno: ~$2 en tokens (40% del pago)
📈 Margen bruto: ~60%
🔄 Usuarios recurren: ~cada mes

⚠️ Nota técnica: Precios basados en tarifas oficiales DeepSeek. deepseek-v4-flash: $0.14/M input, $0.28/M output. deepseek-v4-pro (75% descuento): $0.435/M input, $0.87/M output. El descuento del 75% en v4-pro expira el 31 de Mayo 2026 — después los precios se multiplican ×4. Los modelos deepseek-chat y deepseek-reasoner están deprecados; lv-zero ya migra a deepseek-v4-flash y deepseek-v4-pro.

¿Hacia Dónde Va lv-zero?

La meta no es "ser como Claude Code" o "ser como Antigravity". La meta es superarlos en lenguaje natural.

🎯 lv-zero hará todo lo que Claude Code + Antigravity hacen — pero mejor y en lenguaje natural
